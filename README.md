# SauceDemo Automation Framework

A production-quality Selenium 4 + PyTest automation framework for [SauceDemo](https://www.saucedemo.com/) following the **Page Object Model** pattern. Includes JMeter performance tests.

---

## Prerequisites

| Requirement       | Version   |
|-------------------|-----------|
| Python            | 3.11+     |
| Google Chrome     | Latest    |
| pip               | Latest    |
| Apache JMeter     | 5.6+      |

---

## Setup

```bash
# 1. Clone / navigate to the project
cd d:\st-testing-casestudy

# 2. (Optional) Create a virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux

# 3. Install dependencies
pip install -r requirements.txt
```

> **Note**: ChromeDriver is managed automatically by `webdriver-manager` — no manual download needed.

---

## Running Tests

### Full Suite
```bash
pytest
```

### Smoke Tests Only (Mandatory FRs)
```bash
pytest -m smoke
```

### Regression Tests Only
```bash
pytest -m regression
```

### Headless Mode (faster, no browser window)
```bash
pytest --headless
```

### Generate HTML Report
```bash
pytest --html=reports/report.html --self-contained-html
```
> The HTML report is generated by default via `pytest.ini` settings.

## JMeter Performance Tests

### Run in GUI Mode (debugging)
```bash
jmeter -t performance/jmx/saucedemo_weekend_lab.jmx
```

### Run in Non-GUI Mode (actual test)
```bash
jmeter -n -t performance/jmx/saucedemo_weekend_lab.jmx -l performance/results/results.jtl -e -o performance/html_report/
```

### NFR Requirements

| NFR ID  | Requirement                    | Threshold          |
|---------|--------------------------------|--------------------|
| NFR-01  | Page load under normal load    | ≤ 3 seconds        |
| NFR-02  | 20 concurrent users            | Error rate < 1%    |
| NFR-03  | Response time under peak load  | ≤ 5s (95th pctile) |

See `performance/Findings.md` for detailed results after running the tests.

## Key Design Decisions

1. **Function-scoped fixtures** — each test gets a fresh browser, preventing state leakage
2. **Explicit waits only** — no `time.sleep()` anywhere in the codebase
3. **Automatic screenshot on failure** — via `pytest_runtest_makereport` hook, attached to HTML report
4. **External test data** — CSV for login, JSON for checkout (per requirements)
5. **Structured logging** — Python `logging` module → `reports/logs/test_run.log`
